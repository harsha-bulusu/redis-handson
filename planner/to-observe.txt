Redis uses efficient internal data representations depending on size and structure:

----------------------------------------------------------------------------
|Data Type |Encoding(s)	   |    Description                                 |
-----------------------------------------------------------------------------
STRING	    raw, int	        int used for small integers
LIST	    quicklist	        Compact linked list
SET	        intset, hashtable	Small sets as intset, else hashmap
HASH	    ziplist, hashtable	Compact when <512 entries or small fields
ZSET	    ziplist, skiplist	Sorted set uses ziplist if small
-----------------------------------------------------------------------------
